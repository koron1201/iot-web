# 要件定義

## 概要

スマートICTソリューション研究室Webサイトプロジェクトの要件定義書です。

## プロジェクト概要

### プロジェクト名
スマートICTソリューション研究室Webサイト

### プロジェクト目的
研究室の情報発信、研究成果の公開、メンバー紹介、イベント情報の提供を目的としたWebサイトの構築

### プロジェクト範囲
- 研究室情報の公開
- 研究成果の公開
- メンバー紹介
- イベント・セミナー情報の提供
- お問い合わせ機能
- 管理機能

## 機能要件
### 0. UI骨組み（フロントエンド共通）
- **機能**: サイト全体のUI骨組み（ページ共通レイアウト/ナビ/フッター/ヒーロー/基礎コンポーネント）
- **詳細**:
  - レイアウト: ヘッダー（グローバルナビ）、メイン、フッター
  - ナビ項目: 別紙（docs内）に準拠（本要件では省略）
  - 3Dキャンバス: ヒーローセクションに1シーン（操作は回転/ズーム/ドラッグ対応）
  - 2Dアニメーション: スクロールやフェード等の基本トランジション
  - レスポンシブ: 低優先。破綻防止の最小限ブレークポイントのみ対応
- **優先度**: 高
- **受け入れ基準**:
  - 主要テンプレート（トップ/汎用ページ）で共通レイアウトが動作
  - 3D操作（回転/ズーム/ドラッグ）がデスクトップで正常に機能
  - アクセシビリティ破綻がない（キーボードフォーカス移動可）

## 非機能要件

### 1. パフォーマンス要件

#### 1.1 レスポンス時間
- **要件**: ページ表示時間は3秒以内
- **詳細**: 
  - ホームページ: 2秒以内
  - 一覧ページ: 3秒以内
  - 詳細ページ: 3秒以内
  - 検索結果: 5秒以内
- **測定方法**: ブラウザの開発者ツールで測定
- **優先度**: 高

#### 1.3 3D表示パフォーマンス
- **要件**: デスクトップで快適に動作し、モバイルはフォールバック検討
- **詳細**:
  - 目安: 60fpsを目標（最低30fps維持）
  - モバイル/低性能端末: 静止画・動画等のフォールバックを許容
- **測定方法**: ブラウザプロファイラ/計測用メトリクス
- **優先度**: 中

#### 1.2 スループット
- **要件**: 同時接続数100ユーザーに対応
- **詳細**:
  - 同時接続数: 100ユーザー
  - 1日あたりのアクセス数: 1,000アクセス
  - ピーク時のアクセス数: 200アクセス/時間
- **測定方法**: 負荷テストツールで測定
- **優先度**: 中

### 2. 可用性要件

#### 2.1 稼働率
- **要件**: 稼働率99%以上
- **詳細**:
  - 月間ダウンタイム: 7.2時間以内
  - 年間ダウンタイム: 87.6時間以内
  - 計画メンテナンス: 月1回、2時間以内
- **測定方法**: 監視ツールで測定
- **優先度**: 高

#### 2.2 障害復旧時間
- **要件**: 障害復旧時間は4時間以内
- **詳細**:
  - 軽微な障害: 1時間以内
  - 中程度の障害: 2時間以内
  - 重大な障害: 4時間以内
- **測定方法**: 障害発生から復旧までの時間
- **優先度**: 高

### 3. セキュリティ要件

#### 3.1 認証・認可
- **要件**: 適切な認証・認可機能の実装
- **詳細**:

- **測定方法**: セキュリティテストで確認
- **優先度**: 高

#### 3.2 データ保護
- **要件**: 個人情報の適切な保護
- **詳細**:
  - 個人情報の暗号化
  - アクセスログの記録

  - データ削除機能
- **測定方法**: セキュリティ監査で確認
- **優先度**: 高

### 4. 拡張性要件

#### 4.1 機能拡張
- **要件**: 新機能の追加が容易
- **詳細**:
  - モジュール化された設計

## 制約事項

### 1. 技術的制約
- **ブラウザ対応**: Chrome, Firefox, Safari, Edge の最新版
- **モバイル対応**: レスポンシブデザイン対応
- **OS対応**: Windows, macOS, Linux
- **データベース**: PostgreSQL（推奨）
 - **フロントエンド構成**: SPA（Vite + React）
 - **UIライブラリ**: shadcn/ui（第一候補）
 - **3D操作**: 回転/ズーム/ドラッグ対応

### 2. 運用制約
- **サーバー環境**: Vercel（フロントエンド）、Railway/Heroku（バックエンド）
- **ネットワーク**: HTTPS必須、CDN対応
- **バックアップ**: 自動バックアップ（日次）
- **監視**: Vercel Analytics、Sentry（エラー監視）

### 3. 法的制約
- **個人情報保護**: 個人情報保護法に準拠
- **著作権**: 著作権法に準拠
- **セキュリティ**: 情報セキュリティポリシーに準拠
- **アクセシビリティ**: WCAG 2.1 AA準拠
