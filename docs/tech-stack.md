# 技術スタック

## 概要

スマートICTソリューション研究室Webサイトプロジェクトの技術スタック選定書です。

## 技術スタック選定方針

### 選定基準
- **開発効率**: 開発速度と生産性
- **保守性**: コードの保守性と可読性
- **拡張性**: 将来の機能拡張への対応
- **パフォーマンス**: アプリケーションの性能
- **セキュリティ**: セキュリティ要件への対応
- **学習コスト**: チームメンバーの習得コスト
- **エコシステム**: コミュニティとライブラリの充実度

### 選定プロセス
1. 要件分析
2. 技術調査
3. プロトタイプ作成
4. 評価・比較
5. 最終選定

## フロントエンド技術スタック

### フレームワーク・ライブラリ
- **メインフレームワーク**: React（決定）
  - **候補**: React, Vue.js, Angular, Svelte
  - **選定理由**: Viteとの相性と開発者経験
  - **バージョン**: 19 系（`react`/`react-dom`）

### 言語・ツール
- **プログラミング言語**: TypeScript（決定）
  - **バージョン**: 5.9 系

### ビルドツール
- **ビルドツール**: Vite（決定）
  - **バージョン**: 7 系
  - **備考**: `@vitejs/plugin-react` 使用

### スタイリング
- **CSSフレームワーク**: Tailwind CSS（決定）
  - **バージョン**: 3.4 系（PostCSS 8, Autoprefixer 10）
  - **備考**: `tailwindcss-animate` 導入済み。将来 shadcn/ui 追加を想定

### UIコンポーネント
- **UIライブラリ**: shadcn/ui（第一候補）
  - **備考**: 今回は未導入。Tailwind v3 前提のため互換性あり

### 状態管理
- **状態管理**: Zustand（推奨）
  - **候補**: Redux, Zustand, Recoil, Context API
  - **選定理由**: 軽量で学習コストが低く、TypeScriptとの相性が良い
  - **バージョン**: 4系

### ルーティング
- **ルーティング**: React Router（決定）
  - **バージョン**: 7 系

### HTTPクライアント
- **HTTPクライアント**: TanStack Query（React Query）
  - **候補**: Axios, Fetch API, React Query, SWR
  - **選定理由**: キャッシュ機能とサーバー状態管理が優秀
  - **バージョン**: 5系

### フォーム管理
- **フォーム管理**: React Hook Form + Zod
  - **候補**: React Hook Form, Formik, Yup, Zod
  - **選定理由**: パフォーマンスが良く、Zodとの統合で型安全なバリデーションが可能
  - **バージョン**: React Hook Form 7系、Zod 3系

### テスト
- **テストフレームワーク**: Vitest + Testing Library
  - **候補**: Jest, Vitest, Cypress, Playwright
  - **選定理由**: Viteとの統合が良く、高速なテスト実行が可能
  - **バージョン**: Vitest 1系、Testing Library 14系

### アニメーション
- **2Dアニメーション**: Framer Motion（決定）
  - **備考**: 複雑なタイムラインが必要な場合はGSAPを検討

### 3D/Canvas
- **ライブラリ**: react-three-fiber + drei（決定）
  - **操作**: 回転/ズーム/ドラッグ対応（要件）
  - **モデル形式**: GLTF/GLB（推奨）
  - **スクロール連動**: 検討中（パフォーマンス次第）
  - **フォールバック**: 静止画・動画（モバイル向け）

## バックエンド技術スタック

### フレームワーク・ランタイム
- **ランタイム**: Node.js（推奨）
  - **候補**: Node.js, Python, Java, Go
  - **選定理由**: フロントエンドとの技術統一、豊富なエコシステム
  - **バージョン**: 20系（LTS）

### Webフレームワーク
- **Webフレームワーク**: Fastify（推奨）
  - **候補**: Express.js, Fastify, NestJS, Django, Flask, Spring Boot
  - **選定理由**: 高性能、TypeScriptサポート、プラグインシステム
  - **バージョン**: 4系

### プログラミング言語
- **プログラミング言語**: TypeScript
  - **候補**: TypeScript, JavaScript, Python, Java, Go
  - **選定理由**: フロントエンドとの統一、型安全性
  - **バージョン**: 5系

### データベース
- **データベース**: PostgreSQL
  - **候補**: PostgreSQL, MySQL, MongoDB, SQLite
  - **選定理由**: 高性能、JSONサポート、ACID準拠
  - **バージョン**: 16系

### ORM・ODM
- **ORM/ODM**: Prisma
  - **候補**: Prisma, TypeORM, Sequelize, Mongoose, SQLAlchemy
  - **選定理由**: TypeScriptファースト、優れた開発者体験
  - **バージョン**: 5系

### 認証・認可
- **認証**: JWT + Passport.js
  - **候補**: JWT, Passport.js, Auth0, Firebase Auth
  - **選定理由**: シンプルで柔軟、セキュリティが高い
  - **バージョン**: jsonwebtoken 9系、passport 0.7系

### API仕様
- **API仕様**: REST + OpenAPI/Swagger
  - **候補**: OpenAPI/Swagger, GraphQL, REST
  - **選定理由**: シンプルで理解しやすく、ドキュメント生成が容易
  - **バージョン**: OpenAPI 3.0

### バリデーション
- **バリデーション**: Zod
  - **候補**: Joi, Yup, Zod, class-validator
  - **選定理由**: TypeScriptとの統合が優秀、型推論が強力
  - **バージョン**: 3系

### テスト
- **テストフレームワーク**: Vitest + Supertest
  - **候補**: Jest, Mocha, Chai, Supertest, Pytest
  - **選定理由**: フロントエンドとの統一、高速なテスト実行
  - **バージョン**: Vitest 1系、Supertest 6系

## インフラ・運用技術スタック

### クラウドプラットフォーム
- **クラウドプラットフォーム**: Vercel（フロントエンド）+ Railway（バックエンド）
  - **候補**: AWS, Google Cloud, Azure, Vercel, Netlify
  - **選定理由**: 開発者体験が優秀、自動デプロイ、スケーラビリティ
  - **バージョン**: 最新版

### コンテナ技術
- **コンテナ技術**: Docker + Docker Compose
  - **候補**: Docker, Kubernetes, Docker Compose
  - **選定理由**: 開発環境の統一、デプロイの簡素化
  - **バージョン**: Docker 24系、Docker Compose 2系

### Webサーバー
- **Webサーバー**: Nginx
  - **候補**: Nginx, Apache, Caddy
  - **選定理由**: 高性能、リバースプロキシ機能、静的ファイル配信
  - **バージョン**: 1.24系

### リバースプロキシ
- **リバースプロキシ**: Nginx
  - **候補**: Nginx, Traefik, HAProxy
  - **選定理由**: Webサーバーと統合、設定が簡潔
  - **バージョン**: 1.24系

### ロードバランサー
- **ロードバランサー**: Nginx（小規模）、AWS ALB（大規模）
  - **候補**: AWS ALB, Nginx, HAProxy
  - **選定理由**: コスト効率、スケーラビリティ
  - **バージョン**: Nginx 1.24系、AWS ALB 最新版

### CDN
- **CDN**: Cloudflare
  - **候補**: CloudFront, Cloudflare, Fastly
  - **選定理由**: 無料プラン、DDoS保護、高速配信
  - **バージョン**: 最新版

### 監視・ログ
- **監視**: Vercel Analytics + Sentry
  - **候補**: Prometheus, Grafana, DataDog, New Relic
  - **選定理由**: 開発者体験、エラー追跡、パフォーマンス監視
  - **バージョン**: 最新版

### ログ管理
- **ログ管理**: Vercel Logs + Sentry
  - **候補**: ELK Stack, Fluentd, CloudWatch Logs
  - **選定理由**: 統合された監視、エラー追跡
  - **バージョン**: 最新版

### バックアップ
- **バックアップ**: Railway自動バックアップ + PostgreSQL定期バックアップ
  - **候補**: AWS Backup, Google Cloud Backup, Azure Backup
  - **選定理由**: 自動化、コスト効率
  - **バージョン**: 最新版
